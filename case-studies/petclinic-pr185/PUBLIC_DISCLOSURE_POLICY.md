# Public Disclosure Policy

## Public exactly

The public case study may name public repositories, PR #185, public commit SHAs, public changed paths, public workflow names and conclusions, the public review finding and resolution, the squash merge method, the merge result SHA, and terminal-state names.

## Public only after sanitization

Receipt structures, decision structures, permit structures, failure logs, authentication-admission explanations, trust-boundary explanations, prompt examples, and terminal readouts may appear only as summaries or synthetic examples.

## Private

Raw human decisions, raw local receipts, internal evidence packets, local runtime directories, operator environment observations, permission payloads, raw authentication responses, mutable run state, and full prompt transcripts are not published.

## Never published

Tokens, cookies, authorization headers, private keys, personal access tokens, credential-manager contents, session secrets, recovery codes, and raw environment secrets are never read for publication and never printed.

## Future changes

Future public evidence changes must preserve non-authorizing examples, keep active grants out of the repository, run disclosure validation, and keep claims evidence-bound.
