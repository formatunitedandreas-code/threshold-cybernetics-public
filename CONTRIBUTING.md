# Contributing

## Reproducibility problems

Use the reproducibility report template. Include tool versions, steps performed, expected result, observed result, and sanitized logs. Do not include secrets.

## Governance findings

Use the governance finding template. Identify the affected state transition, authority boundary, possible failure mode, evidence, and suggested mitigation.

## Documentation changes

Use the documentation problem template. Keep claims evidence-bound and preserve explicit non-claims.

## Security issues

If a finding may expose credentials, authentication context, private receipts, or operator details, treat it as security-sensitive and do not post sensitive material publicly.

## AI-generated contributions

Disclose AI assistance in pull requests. Confirm that public examples remain non-authorizing and that disclosure validation passed.
